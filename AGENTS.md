# AGENTS.md

## Cursor Cloud specific instructions

This is a Vue 3 + Vite+ + TypeScript + Vant 4 mobile H5 starter template. Frontend-only, no backend services required. The toolchain is **Vite+** (the `vp` CLI: dev/build/test/lint/format unified, Rolldown + Oxlint + Oxfmt under the hood). `vite-plus` is pinned to an exact version because it is pre-1.0 — do not loosen it to a `^`/`~` range.

### Quick reference

| Task                          | Command              |
| ----------------------------- | -------------------- |
| Dev server (port 9527)        | `pnpm dev`           |
| Lint (auto-fix)               | `pnpm lint`          |
| Type check                    | `pnpm type-check`    |
| Unit tests                    | `pnpm test`          |
| Tests (watch)                 | `pnpm test:watch`    |
| Coverage                      | `pnpm test:coverage` |
| Build (type-check + vp build) | `pnpm build`         |
| Format (Oxfmt)                | `pnpm format`        |
| Check (format + lint + types) | `pnpm check`         |

See `README.md` for full documentation.

### Gotchas

- **pnpm build scripts**: `esbuild`, `core-js`, and `@parcel/watcher` require build scripts. These are allowed via `pnpm.onlyBuiltDependencies` in `package.json`. If you see "Ignored build scripts" warnings after `pnpm install`, Vite will not work — ensure these packages are in the allow-list.
- **Auto-import type generation**: `auto-imports.d.ts` and `components.d.ts` are generated by `unplugin-auto-import` and `unplugin-vue-components` during build/dev. If type-check fails on auto-imported APIs (`computed`, `ref`, `useRouter`, etc.), run `pnpm build` (or `vp dev`) once to regenerate these files. They are excluded from Oxfmt so they don't churn on every build.
- **Vite+ config**: `vite.config.ts` must export a plain object (not a `({ mode }) => ...` function) — `vp` reads the `lint`/`fmt` fields statically, and `defineConfig` is intentionally omitted because its overloads crash vue-tsc on the large `lint` block. Mode is derived from argv at the top of the file.
- **Styling**: UnoCSS utilities (presetWind3 + preset-rem-to-px) plus the CSS-variable theme in `src/style.css`. Utilities emit rem → px → vw via `postcss-mobile-forever`. Theme colors map to CSS vars (e.g. `text-primary`, `bg-accent-surface`).
- **Git hooks**: `pre-commit` runs `lint-staged` (`vp lint --fix` + `vp fmt` on staged files); `commit-msg` enforces Conventional Commits via commitlint.
- **API proxy**: Dev server proxies `/api` to a placeholder backend (`api.example.com`). The API demo card on the About page uses TanStack Vue Query; clicking it triggers a `refetch` that fails in dev without a real backend (shows "无后端服务") — this is expected.
- **Stores use Composition API**: All Pinia stores use the setup store pattern (Composition API), not Options API.
